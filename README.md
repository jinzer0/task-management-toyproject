# RTDM — Robust Task Deadline Manager

계층적 예외 처리 구조를 갖춘 CLI 기반 태스크 마감일 관리 시스템.

Python 표준 라이브러리만으로 구현되었으며, 데이터 입력부터 파일 입출력까지 발생 가능한 모든 예외를 구조화하여 시스템 중단 없는 안정적인 실행 환경을 제공한다.

## 기능

| 기능 | 설명 |
|------|------|
| 태스크 추가 | 제목, 우선순위(high/medium/low), 마감일(YYYY-MM-DD) 입력 |
| 태스크 수정 | 부분 수정 지원 — 변경할 항목만 입력, 나머지는 Enter로 건너뜀 |
| 태스크 삭제 | 삭제 전 확인 절차 포함 |
| 상태 토글 | 미완료 ↔ 완료 전환 |
| 전체 목록 조회 | ID 순 정렬 |
| 우선순위별 조회 | High → Medium → Low 순 |
| 마감일 임박순 조회 | 가까운 마감일 순, 경과 시 `[기한 경과]` 표시 |

## 프로젝트 구조

```
task/
├── main.py        # CLI 인터페이스 및 메인 루프
├── manager.py     # 비즈니스 로직 및 유효성 검사
├── models.py      # Task 데이터 모델 및 커스텀 예외 정의
├── storage.py     # 파일 I/O 및 JSON 직렬화 (원자적 쓰기)
└── tasks_data.txt # 자동 생성되는 데이터 파일 (JSON)
```

## 설치 및 실행

### 요구 사항

- Python 3.6 이상
- 외부 패키지 불필요 (표준 라이브러리만 사용)

### 실행

```bash
python main.py
```

최초 실행 시 `tasks_data.txt` 파일이 자동 생성된다.

## 예외 처리 구조

`TaskError`를 루트로 하는 계층적 예외 체계를 통해, 모든 오류 상황에서 프로그램이 종료되지 않고 한국어 에러 메시지를 출력한 뒤 메뉴로 복귀한다.

```
TaskError
├── InvalidDateError        — 날짜 형식 오류, 과거 날짜
├── InvalidPriorityError    — 허용되지 않은 우선순위 값
├── InvalidInputError       — 빈 값, 숫자 대신 문자 입력
├── DuplicateTitleError     — 중복 제목
├── TaskNotFoundError       — 존재하지 않는 ID 접근
├── DataCorruptionError     — JSON 파싱 실패, 스키마 불일치
└── StorageError            — 파일 권한 부족, I/O 실패
```

## 데이터 형식

`tasks_data.txt`는 JSON 배열로 저장되며, 수동 편집이 가능한 텍스트 형식이다.

```json
[
  {
    "id": 1,
    "title": "프로젝트 보고서 작성",
    "priority": "high",
    "deadline": "2026-04-01",
    "status": "pending",
    "created_at": "2026-03-13T14:30:00.000000"
  }
]
```

  7. 마감일 임박순 조회

  메뉴 선택: 1

--- 태스크 추가 ---

  태스크가 추가되었습니다.
  [1] 기말고사 준비  | 우선순위: 높음  | 마감일: 2026-04-15  | 상태: 미완료
